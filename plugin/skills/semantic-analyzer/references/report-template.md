# 语义深检报告（输出模板）

执行 STEP 4 时按下列 Markdown 结构填写；将 `{...}` 与示例占位替换为实际研判结果。

```markdown
## 🔬 深度语义分析报告

**来源**: {文件名或文本片段描述}
**分析模式**: 实时语义推理（超越固定词典）

### 发现总览
- 新发现的陷阱词: N 个（词典未收录）
- 上下文风险: M 处
- 整体风险等级: {等级}

### 新发现的语义陷阱词

| # | 词汇 | 四维评分 | 风险等级 | 上下文 | 替换建议 |
|---|------|---------|---------|--------|---------|
| 1 | 优化 | 11/12   | critical | "优化代码结构" | "重构为单一职责模式" |
| 2 | ...  | ...     | ...     | ...    | ...     |

#### 详细评分

**词汇: "优化"**（第 X 行）
- 程度性: 3/3 — 优化是连续性的，没有明确终点
- 展望性: 2/3 — 隐含"变得更好"的未来预期
- 主观评价性: 3/3 — "更好"的标准因人/场景而异
- 关联松散度: 3/3 — 性能/可读性/架构/安全均可被"优化"
- **总分: 11/12 → critical**
- **替换建议**: 将"优化代码结构"替换为具体操作，如"将超过 200 行的函数拆分为独立模块"

### 上下文风险

1. **语义叠加** [L{n}]: "分析代码质量并给出优化建议"
   → 三个宽边界词共现，风险叠加
   → 建议: 拆分为具体的独立指令

2. **隐式宽边界** [L{n}]: "全面检查代码"
   → "全面"让检查范围无限扩大
   → 建议: 替换为"检查代码中的以下 3 种缺陷"

### 操作建议

1. **立即替换**（critical/high）:
   - 第 X 行: "优化" → "重构为..."
   - 第 Y 行: "..."

2. **建议替换**（medium-high/medium）:
   - ...

3. **收录到词典**:
   以下新发现的高频陷阱词建议通过 **lexicon-manager** 收录：先更新 `plugin/references/semantic-trap-lexicon.md` 中对应表格，再运行 `npm run build-lexicon` 与 `npm test`。
```
