# 实时语义陷阱分析器

超越固定词典的限制，利用 Claude 自身的语义理解能力，对指令文本进行深度语义分析，发现词典中尚未收录但可能导致大模型语义发散的词汇。

## 定位：与 Hook 检测的关系

| 层级 | 机制 | 速度 | 覆盖面 |
|------|------|------|--------|
| 第 1 层：词典匹配 | Hook 自动触发，27 个已知词对的 O(1) 查找 | 毫秒级 | 固定，仅限已知词汇 |
| 第 2 层：结构分析 | Hook 自动触发，4 种结构模式的规则匹配 | 毫秒级 | 固定，仅限已知模式 |
| **第 3 层：语义分析** | **本 Skill，按需触发，Claude 实时语义推理** | **秒级** | **动态，可发现任意新陷阱词** |

Hook 是快速防线（已知威胁），本 Skill 是深度扫描（未知威胁）。

## 语义陷阱的四个判定特征

一个词是否为语义陷阱，基于以下四个特征判定：

1. **程度性 vs 二元性**：陷阱词往往是程度性的（高/中/低），而安全词是二元的（是/否、通过/不通过）
2. **展望性**：陷阱词带有前瞻性含义（"潜在的""可能的"），安全词关注当下事实
3. **主观评价性**：陷阱词触发主观判断，安全词触发客观判定
4. **关联网络松散度**：陷阱词在 LLM 语义空间中的关联网络松散且广阔，安全词的关联网络紧密且有限

**核心原理**：满足越多特征的词，语义边界越宽，越容易导致 LLM 输出突破 Prompt 约束。

## 分析工作流

### STEP 1：获取待分析文本

获取用户提供的指令文本。来源可以是：
- 用户直接粘贴的文本片段
- 指定的文件路径（读取文件内容）
- 当前正在编辑的 Skill/Agent/Rule 文件

同时读取 `plugin/lib/lexicon-data.js` 获取已知词典，避免重复报告已收录的词汇。

### STEP 2：逐句深度语义扫描

对文本中的每一个有实质内容的句子（跳过标题、代码块、空行），执行以下分析：

**2a. 提取关键词**

从句子中提取所有可能影响 LLM 行为的词汇，按以下类别分类：

- **动作指令词**：指导 LLM 执行什么操作（动词/动宾短语）
- **目标对象词**：定义操作的目标（名词/名词短语）
- **约束修饰词**：限定范围或程度的词（形容词/副词/情态词）
- **输出描述词**：描述期望输出形式的词

**2b. 对每个关键词进行四维评分**

对每个提取的关键词，按四个特征维度评分（0-3 分）：

```
词汇: "优化"
┌──────────────┬───────┬──────────────────────────────────┐
│ 特征维度     │ 评分  │ 判定理由                          │
├──────────────┼───────┼──────────────────────────────────┤
│ 程度性       │ 3/3   │ 优化是连续性的，没有明确终点      │
│ 展望性       │ 2/3   │ 隐含"变得更好"的未来预期         │
│ 主观评价性   │ 3/3   │ "更好"的标准因人/场景而异         │
│ 关联松散度   │ 3/3   │ 性能/可读性/架构/安全均可被优化   │
├──────────────┼───────┼──────────────────────────────────┤
│ 总分         │ 11/12 │ → 高危陷阱词                     │
└──────────────┴───────┴──────────────────────────────────┘
```

**评分标准**：
- 0 分：完全不具备该特征
- 1 分：弱程度具备
- 2 分：中等程度具备
- 3 分：强烈具备

**风险等级映射**：
- 10-12 分：critical（极高危）
- 7-9 分：high（高危）
- 5-6 分：medium-high（中高危）
- 3-4 分：medium（中危）
- 0-2 分：low（低危，可忽略）

**2c. 排除已知词汇**

如果该词已在 `lexicon-data.js` 的词典中（wideWordsZh 或 wideWordsEn），跳过（已由 Hook 层覆盖）。

**2d. 生成窄边界替换建议**

对于评分 ≥ 5 的词汇，提出具体的窄边界替换词。替换词应满足：
- 在当前句子上下文中语义等价
- 四维评分显著低于原词
- 不引入新的歧义

### STEP 3：上下文风险增强分析

除了逐词分析，还需检测以下上下文级别的风险模式：

**3a. 语义叠加效应**

当多个中等风险词在同一句中共现时，风险会叠加放大：
```
"分析代码质量并给出优化建议"
 ^^^^          ^^^^  ^^^^
 中高危  +    高危  + 中高危  = 整句风险极高
```

**3b. 隐式宽边界**

某些词组表面无害，但组合后产生宽边界效应：
- "全面检查" — "全面"让"检查"的边界无限扩大
- "深入分析" — "深入"触发层层递进的推理链
- "仔细审视" — "仔细"让模型过度关注细节

**3c. 否定句中的语义反转**

否定句中使用宽边界词，风险不降反升：
```
"不要忽略任何潜在风险"
→ LLM 会努力"不忽略"，反而激活更多联想
```

### STEP 4：生成分析报告

按以下格式输出：

```markdown
## 🔬 深度语义分析报告

**来源**: {文件名或文本片段描述}
**分析模式**: 实时语义推理（超越固定词典）

### 发现总览
- 新发现的陷阱词: N 个（词典未收录）
- 上下文风险: M 处
- 整体风险等级: {等级}

### 新发现的语义陷阱词

| # | 词汇 | 四维评分 | 风险等级 | 上下文 | 替换建议 |
|---|------|---------|---------|--------|---------|
| 1 | 优化 | 11/12   | critical | "优化代码结构" | "重构为单一职责模式" |
| 2 | ...  | ...     | ...     | ...    | ...     |

#### 详细评分

**词汇: "优化"**（第 X 行）
- 程度性: 3/3 — 优化是连续性的，没有明确终点
- 展望性: 2/3 — 隐含"变得更好"的未来预期
- 主观评价性: 3/3 — "更好"的标准因人/场景而异
- 关联松散度: 3/3 — 性能/可读性/架构/安全均可被"优化"
- **总分: 11/12 → critical**
- **替换建议**: 将"优化代码结构"替换为具体操作，如"将超过 200 行的函数拆分为独立模块"

### 上下文风险

1. **语义叠加** [L{n}]: "分析代码质量并给出优化建议"
   → 三个宽边界词共现，风险叠加
   → 建议: 拆分为具体的独立指令

2. **隐式宽边界** [L{n}]: "全面检查代码"
   → "全面"让检查范围无限扩大
   → 建议: 替换为"检查代码中的以下 3 种缺陷"

### 操作建议

1. **立即替换**（critical/high）:
   - 第 X 行: "优化" → "重构为..."
   - 第 Y 行: "..."

2. **建议替换**（medium-high/medium）:
   - ...

3. **收录到词典**:
   以下新发现的高频陷阱词建议收录到 lexicon-data.js:
   - "优化" (critical) — 窄边界替换: 视上下文而定
   - ...
   可使用 `lexicon-manager` skill 执行收录。
```

### STEP 5：与用户交互

报告生成后：

1. 向用户展示完整报告
2. 询问是否需要：
   - 直接应用替换建议（编辑文件）
   - 将新发现的高频陷阱词收录到词典（调用 lexicon-manager）
   - 对某个词的判定进行讨论或调整

## 常见高危模式速查

以下是词典之外常见的隐式陷阱模式，分析时优先关注：

### 动词类
- "优化/Optimize" — 无明确终点，触发无限改进
- "改进/Enhance" — 同上
- "探索/Explore" — 触发发散性搜索
- "考虑/Consider" — 触发多角度权衡
- "确保/Ensure" — 触发过度防御性行为
- "处理/Handle" — 边界极宽，几乎可包含任何操作

### 修饰词类
- "全面/Comprehensive" — 让任何操作的范围无限扩大
- "深入/In-depth" — 触发递进式推理链
- "适当/Appropriate" — 主观判断，因语境而异
- "合理/Reasonable" — 同上
- "相关/Relevant" — 关联性由 LLM 自行判断

### 名词类
- "质量/Quality" — 多维度，无统一标准
- "性能/Performance" — 可指速度/内存/并发等多个维度
- "安全/Security" — 涵盖极广的领域
- "最佳实践/Best Practice" — 触发大量通用建议

## 注意事项

- 本 Skill 是深度分析工具，适合对关键指令文件做细致审查
- Hook 层的词典检测仍然是第一道防线，本 Skill 是补充而非替代
- 新发现的高频陷阱词应通过 lexicon-manager 收录到词典，提升 Hook 层的自动检测能力
- 四维评分存在主观性，对于争议词汇应与用户讨论确认
